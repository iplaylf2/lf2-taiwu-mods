## 📦 非托管供应商依赖 (Unmanaged Vendor Dependencies)

此目录用于管理和打包那些不由官方或社区发布在公共 NuGet 源上的第三方依赖项。

### 🎯 目的与实现方式

**目的**：
《太吾绘卷》的部分核心程序集（例如 `Taiwu.Backend.dll`, `Taiwu.Frontend.dll`）以及一些项目依赖的 Unity 包（如 `UniTask`）没有官方的 NuGet 包。为了在 C# 项目中方便地进行版本管理和依赖引用，我们在此处将这些 DLL 文件手动打包成 NuGet 包，并发布到私有的 GitHub Packages 源。

**实现方式**：
此目录下的项目根据依赖的类型，采取了两种不同的打包策略：

#### 1. 游戏核心程序集 (仅编译时依赖)
位于 `game/` 目录下的项目被创建为**仅编译时（compile-time only）的引用程序集**。这意味着你的 Mod 项目在编译时可以链接这些库，但在运行时，它期望游戏环境本身来提供这些 DLL 文件。

#### 2. UPM 及其他第三方库 (运行时依赖)
位于 `upm/` 目录下的项目被创建为**标准的运行时依赖**，其 DLL 文件会随你的 Mod 一同发布和运行。

这类库（如 UniTask）通常是为 Unity 包管理器（UPM）设计的，需要在 Unity IDE 中使用，因此缺少官方的 NuGet 发布源。为了确保兼容性，它们的版本也需要与游戏使用的 Unity 版本及库版本严格对齐。

> **💡 提示**
> 对于运行时依赖，为了避免与其他 Mod 产生 DLL 冲突，最佳实践是使用 `ILRepack` 工具将它们内嵌到你的主程序集中。

---

### ⚠️ 重要风险提示

**警告：使用此目录打包和分发第三方库（尤其是闭源游戏文件）涉及重大的法律与技术风险。**

1.  **法律风险 (Legal Risk)**
    *   直接引用和重新分发游戏的核心 DLL 文件可能违反游戏的最终用户许可协议（EULA）。尽管这些包主要用于私有源上的 Mod 开发，但该行为本身仍处于法律灰色地带。

2.  **技术风险 (Technical Risk)**
    *   **版本兼容性**：游戏本体更新会导致手动打包的 DLL 过时，并可能破坏依赖于这些包的 Mod。
    *   **维护负担**：每次游戏更新都必须重复“提取-打包-发布”的循环，以保持依赖最新。

**总之，此目录提供了一种在没有官方支持下管理依赖的变通方案，但它本质上是脆弱且有风险的。**
